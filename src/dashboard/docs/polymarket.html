Overview

The Polymarket system gives AI agents the ability to trade prediction markets on Polymarket — the world's largest prediction market platform. It combines:

  • 108 specialized tools across 9 skills — trading, on-chain analysis, social intelligence, event feeds, advanced analytics, smart execution, counter-intelligence, portfolio management, and quantitative modeling
  • Real-time on-chain intelligence — whale tracking, orderbook depth, net flow analysis, wallet profiling, liquidity mapping
  • Social sentiment analysis — Twitter, Reddit, Telegram, Polymarket comments, cross-platform velocity detection
  • Smart execution — sniper orders, TWAP/VWAP scale-in, correlation hedging, automated exit strategies
  • Counter-intelligence — manipulation detection, resolution risk scoring, counterparty analysis
  • 14 quant tools — Kelly criterion, Black-Scholes, Bayesian inference, Monte Carlo simulation, regime detection
  • Self-improving agents — prediction journaling, calibration tracking, strategy performance, P&L attribution
  • Full dashboard — 17 tabs across 5 groups (Core, Intelligence, Execution, Learning, Portfolio)
  • Auto-SDK install — the Polymarket CLOB SDK installs automatically, zero manual setup
  • DB persistence — 25+ tables for config, trades, signals, whales, events, predictions, calibration
  • Dedicated system prompt — 6-phase trading loop, 10 risk commandments, tool choreography

Everything goes through the standard enterprise permission pipeline — risk levels, side-effect checks, approval queues, guardrails, and audit logging.

Getting Started

1. Create a Polymarket Agent

Go to Agents → Create Agent and select the "Polymarket Trader" template from the Finance category. This pre-assigns all 9 skills (108 tools). Or assign skills individually:

SkillToolsPurpose
polymarket63Trading infrastructure, learning
polymarket-quant14Mathematical analysis engine
polymarket-onchain6Blockchain intelligence
polymarket-social5Social media sentiment
polymarket-feeds5Event calendar, news, odds
polymarket-analytics5Correlation, arbitrage, regime
polymarket-execution4Smart order execution
polymarket-counterintel3Manipulation detection
polymarket-portfolio3Portfolio optimization

2. Connect a Wallet

The agent needs an Ethereum wallet with USDC on Polygon. Two options:

  • Auto-create: poly_create_account method="auto" — generates a fresh wallet
  • Import: poly_create_account method="import" — use your existing private key

3. Configure Risk Limits

Use the Config tab on the dashboard or poly_set_config. Set trading mode, max position, daily loss limit, drawdown threshold.

4. Fund & Start

Send USDC to the wallet on Polygon. Start with paper trading mode to test strategies risk-free.

Performance

  • Shared response cache — all external API calls cached with configurable TTL (15s–60s)
  • Rate limiting — 30 requests/minute per domain to prevent API bans
  • Parallel fetches — Promise.all for multi-token analysis, cross-platform sentiment
  • Retry logic — automatic retry with exponential backoff for transient failures
  • Config reads from DB — no stale in-memory state; dashboard config changes take effect on next tool call

Learning System (8 tools)

The Learning Loop

  1. Recall: poly_recall_lessons — check past lessons for this market type
  2. Predict: poly_record_prediction — journal prediction with reasoning BEFORE trading
  3. Execute: poly_place_order + poly_exit_strategy
  4. Resolve: poly_resolve_prediction — log outcome → auto-updates calibration
  5. Review: poly_trade_review → poly_record_lesson — extract insights
  6. Calibrate: poly_calibration — "Am I overconfident at 80%?"
  7. Rank: poly_strategy_performance — which signals make money?
  8. Attribute: poly_pnl_attribution — P&L by strategy/category/signal

System Prompt

Agents with Polymarket skills get an injected system prompt (buildPolymarketPrompt()) containing:

  • 6-phase trading loop: Scan → Analyze → Decide → Execute → Monitor → Learn
  • 10 risk commandments: Never risk >5% on one market, never trade without checking liquidity, always set exit rules, etc.
  • Anti-patterns: Don't revenge trade, don't chase, don't anchor to entry price
  • Tool choreography: Which tools to use together for each strategy type
  • Category-specific edge sources: What signals work for politics, sports, crypto, economics, legal, science

Best Practices

  • Start with paper trading — test strategies before real money
  • Always run pre-trade checks — resolution_risk + manipulation_detector + microstructure + recall_lessons
  • Use scale-in for large orders — $500+ in thin markets should use TWAP/VWAP
  • Set exit rules immediately — every position needs take-profit, stop-loss, or trailing stop
  • Check calibration weekly — if overconfident, reduce position sizes
  • Review P&L attribution monthly — kill strategies with <45% win rate over 20+ trades
  • Track whale wallets — add consistently profitable wallets to your watchlist
  • Monitor social velocity — spikes of 3x+ acceleration precede market moves
  • Use hedges for large positions — reduce directional risk via correlated markets
  • Record every prediction — the learning system only works with data
